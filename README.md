# AtrocityViz

Standalone visualization scripts and Shiny apps extracted from the main AtrocityNet project.

Structure
- `01_data/`: small bundled app data for local/source use
- `02_scripts/static/`: static network construction and plotting scripts
- `02_scripts/shiny/`: interactive Shiny apps
- `02_scripts/deploy/`: deployment-ready app bundles
- `03_results/`: local outputs generated by AtrocityViz workflows
- `.github/workflows/`: GitHub Pages deployment workflow for the circular Shiny app

Current data expectation
- Each script resolves the project root on its own and reads bundled input files from `01_data/`.
- You can override the input file path for a run by setting `ATROCITYVIZ_DATA` in your R session before sourcing a script.

Current output locations
- Static scripts write figure files directly into `03_results/`.
- Static scripts no longer write `.rds` graph objects.

Posit / Shiny note
- These Shiny apps are plain local Shiny apps.
- They are not linked to Positron Cloud, `rsconnect`, or `shinyapps.io`.
- To publish them later, you would need to add deployment configuration separately.

Current deployment bundle
- `02_scripts/deploy/shiny_circular_conflict/` contains the previously uploaded Posit Cloud app bundle:
  - `app.R`
  - `ged_2025.csv`

GitHub Pages deployment
- `.github/workflows/deploy-circular-shiny-pages.yml` publishes the circular app bundle to GitHub Pages using `shinylive`.
- `02_scripts/deploy/export_circular_shinylive.R` exports the bundle locally to `_site/` for preview or manual checks.
- One-time GitHub setup: in repo `Settings` -> `Pages`, set the source to `GitHub Actions`.
- After that, pushes touching the circular deploy bundle or the workflow will publish the app automatically.
